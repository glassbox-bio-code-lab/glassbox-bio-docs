# Glassbox Bio Documentation

Welcome to the Glassbox Bio docs hub.

## Start here

Use this docs site according to where you are in the workflow:

1. **Getting Started** if you are new to the platform.
2. **PreFlight UI** if you are verifying inputs before submission.
3. **Computational Safety Diligence** if you are running or interpreting core analysis.
4. **Modules** if you are using optional capability layers such as IP/FTO.
5. **Deployment & Operations** if you are setting up or administering the platform.

## Main workflows

### Computational Safety Diligence
The core analysis and reporting workflow. This section explains how to prepare, submit, interpret, and operationalize computational safety diligence outputs.

### PreFlight UI
The pre-submission validation workflow. This section explains input verification, readiness checks, and handoff into analysis.

### Modules
Optional extensions layered onto the core workflow. Each module defines its own required inputs and explains how its outputs interact with the core decision flow.

## Suggested reading path

- New user: Getting Started → PreFlight UI → Computational Safety Diligence
- Technical operator: Deployment & Operations → PreFlight UI → Reference
- Decision-maker: Getting Started → Computational Safety Diligence → Trust & Security
