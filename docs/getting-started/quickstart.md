# Quickstart

This page should eventually become your most-used onboarding document.

## Goal

Get a new user from zero to a valid, interpretable first run as quickly as possible.

## Suggested quickstart flow

1. Prepare a small representative input set.
2. Run it through PreFlight UI.
3. Fix any blocking validation issues.
4. Submit to Computational Safety Diligence.
5. Review the report package and evidence-linked outputs.
6. Decide whether to iterate, escalate, or export.

## Keep this page narrow

Do not overload the quickstart with every concept. Link outward instead.
